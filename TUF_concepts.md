# 🧠 How to Practice Effectively

For every problem:

---

## ✅ Step 1 — Understand the Pattern

Ask yourself:

- Which pattern does this belong to?
- Can I solve it using brute force first?
- How can I optimize it?

---

## ✅ Step 2 — Solve in 3 Levels

### 1️⃣ Brute Force

- Focus on correctness
- Understand the logic clearly

### 2️⃣ Better Solution

- Reduce unnecessary operations
- Improve time complexity

### 3️⃣ Optimal Solution

- Best possible approach
- Best time + space complexity balance

---

# 🎯 Interview Thinking Framework

For every important problem, practice answering these:

---

## ▶️ Why did you solve it this way?

Explain:

- Why this approach works
- Why it is better than brute force
- Which pattern helped you identify the solution

---

## ▶️ What would you change if traffic grows 10x?

Think about:

- Time complexity optimization
- Space optimization
- Scalability
- Caching / preprocessing if needed

---

## ▶️ Can you debug this live?

Practice debugging:

- Dry runs
- Edge cases
- Pointer movement
- Off-by-one errors
- Infinite loops
- Boundary conditions

---

## ▶️ What trade-offs did you make?

Examples:

| Trade-off                 | Reason           |
| ------------------------- | ---------------- |
| Extra space for HashMap   | Faster lookup    |
| Sorting before processing | Easier traversal |
| Recursion vs Iteration    | Cleaner logic    |

---

## ▶️ How did you test this?

Always test:

### ✅ Edge Cases

- Empty input
- Single element
- Duplicate values
- Negative numbers
- Large inputs
- Already sorted arrays
- Reverse sorted arrays
